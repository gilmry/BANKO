API Endpoints Summary
====================

All endpoints under: /api/v1/mobile
All require: Authorization: Bearer <token> (except login)

Authentication (7 endpoints):
  POST   /devices                    - Register device
  GET    /devices                    - List devices
  DELETE /devices/{id}               - Deactivate device
  POST   /devices/{id}/pin           - Set PIN
  POST   /devices/{id}/biometric     - Enable biometric
  POST   /auth/login                 - Login (PIN/biometric)
  POST   /auth/refresh               - Refresh session

Account (3 endpoints):
  GET    /dashboard?locale=en|fr|ar  - Get dashboard
  GET    /offline-cache              - Get offline data
  POST   /sync                       - Sync changes

Payments (3 endpoints):
  POST   /payments/transfer          - Quick transfer
  GET    /payments/beneficiaries     - Get top 5 beneficiaries
  POST   /payments/scan-qr           - Scan QR code

Key Features
============

✅ Device Management
   - Up to 5 devices per customer
   - PIN (4-6 digits, bcrypt hashed)
   - Biometric enablement
   - Remote device deactivation
   - Push token support

✅ Mobile Sessions
   - 30-minute TTL
   - Refresh token rotation
   - Per-device tracking

✅ Dashboard Optimization
   - Single API call aggregates all data
   - 80% reduction in HTTP roundtrips
   - Locale-aware greetings (en, fr, ar)
   - Time-based greeting logic

✅ Offline Support
   - 24-hour cached data
   - Recent transactions
   - Incremental sync on reconnect

✅ Mobile Payments
   - Quick transfers
   - Auto-detect IBAN or phone
   - 2FA for amounts > 1000 TND
   - Frequent beneficiaries
   - QR code scanning

✅ Security
   - PIN validation via bcrypt
   - Biometric validation per device
   - Session tokens with expiry
   - Push token encryption
   - No hardcoded secrets
