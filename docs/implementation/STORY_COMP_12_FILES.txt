================================================================================
KEY VALIDATION RULES (Domain Layer)
================================================================================

BiometricVerification Validation:

1. Construction:
   - validity_days must be > 0
   - Otherwise: Err(DomainError::InvalidState)

2. State Transitions:
   - Pending -> InProgress: allowed
   - InProgress -> Verified or Failed: allowed
   - Other transitions: Err(DomainError::InvalidStateTransition)

3. Verification Completion:
   - confidence_score must be in [0.0, 1.0]
   - If confidence >= 0.95 AND liveness_check = true:
     Status becomes Verified, verified_at is set
   - Otherwise: Status becomes Failed

4. Validity Check:
   - is_valid() returns true only if:
     * status == BiometricStatus::Verified
     * AND Utc::now() < expires_at
