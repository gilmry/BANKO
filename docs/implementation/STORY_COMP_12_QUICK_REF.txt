KEY FEATURES
================================================================================

Verification Lifecycle:
  Pending -> InProgress -> Verified (if confidence>=0.95 && liveness=true)
                        -> Failed (if confidence<0.95 || liveness=false)
                        -> Expired (when expires_at is reached)

Confidence Scoring:
  - 0.0 to 1.0 scale
  - Minimum threshold: 0.95 for verification success
  - Configurable per implementation

Liveness Detection:
  - Boolean flag tracking anti-spoofing checks
  - Required for successful verification
  - Prevents replay attacks with static photos

Document Verification:
  - Support for ID documents (CIN, Passport, etc.)
  - OCR capability via document_scan type
  - Document number storage for audit

Expiration Management:
  - Configurable validity (default 90 days)
  - is_valid() checks status + expiration
  - Automatic expiration detection

VALIDATION RULES
================================================================================

Construction:

State Transitions:
  ✓ Pending -> InProgress (only from Pending)
  ✓ InProgress -> Verified or Failed (only from InProgress)
  ✓ Any -> Expired (when expires_at is reached)

Verification Completion:
  ✓ confidence_score in [0.0, 1.0]
  ✓ Verified if: confidence >= 0.95 AND liveness_check = true
  ✓ Failed if: confidence < 0.95 OR liveness_check = false

Validity Check (is_valid):
  ✓ status == Verified
  ✓ Utc::now() < expires_at
